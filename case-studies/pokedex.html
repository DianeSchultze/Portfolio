Pokedex

A CRT monitor inspired Pokedex built with PokéAPI

GitHub Repo Live Site

The Problem

The goal was to create a small interactive web application that could pull Pokémon data dynamically while still feeling visually distinctive and engaging. The challenge was balancing API-driven functionality with a strong themed interface.

The Solution

I built a front-end Pokedex using HTML, CSS, and JavaScript, powered by PokéAPI. The interface was designed with a CRT monitor inspired visual style to give the project a more unique and nostalgic feel while keeping the experience clear and easy to use.

Tech Stack

  • HTML5 for structure
  • CSS3 for styling and theming
  • JavaScript for dynamic rendering
  • PokéAPI for external data

Key Features

  • API-powered Pokémon data
  • Dynamic content rendering
  • Search and exploration interface
  • CRT-inspired visual direction

Process

This project focused on working with external API data in a front-end-only environment. A key part of the process was structuring how data from PokéAPI would be fetched, handled, and displayed in a way that remained clear and responsive.

The visual direction was intentionally styled around a CRT monitor concept to create a more memorable and themed experience. This helped transform a simple data-driven app into something more engaging while still maintaining usability.

The build also served as a strong exercise in JavaScript fundamentals, including working with asynchronous data, rendering UI dynamically, and managing user interaction.

Outcome

The final result is a polished, interactive web application that combines playful design with technical execution. It highlights my ability to work with APIs, build dynamic interfaces, and create visually distinct front-end experiences.
